Browsers

<div><p>By default, Guix does not have many options for browsers. However, with nonguix, Nix, Flatpak, or Docker, you have many more options. Nonfree options are indicated with a bold <strong>NF</strong> next to the heading, and this includes things not in the official Guix repositories. </p> <p>If you have yet to set up Nix, and would like to, please visit the "Nix" page &lt;placeholder&gt;. Nix is the recommended way to install most nonfree software over Flatpak and Docker (allows for maintaining a declarative configuration.) </p> <h2><a id="table-of-contents" class="anchor" href="#table-of-contents">¶</a>Table of Contents</h2><ul><li><a href="#icecat">Icecat</a> <ul><li><a href="#installation">Installation</a></li> <li><a href="#troubleshooting">Troubleshooting</a></li> </ul></li> <li><a href="#firefox---nf">Firefox - <strong>NF</strong></a> <ul><li><a href="#installation-1">Installation</a></li> <li><a href="#troubleshooting-1">Troubleshooting</a></li> </ul></li> <li><a href="#ungoogled-chromium">Ungoogled Chromium</a> <ul><li><a href="#installation-2">Installation</a></li> <li><a href="#troubleshooting-2">Troubleshooting</a></li> </ul></li> <li><a href="#nyxt">Nyxt</a> <ul><li><a href="#installation-3">Installation</a></li> <li><a href="#troubleshooting-3">Troubleshooting</a></li> </ul></li> <li><a href="#qutebrowser">Qutebrowser</a> <ul><li><a href="#installation-4">Installation</a></li> <li><a href="#troubleshooting-4">Troubleshooting</a></li> </ul></li> <li><a href="#chromium---nf">Chromium - <strong>NF</strong></a> <ul><li><a href="#installation-5">Installation</a></li> <li><a href="#troubleshooting-5">Troubleshooting</a></li> </ul></li> <li><a href="#brave---nf">Brave - <strong>NF</strong></a> <ul><li><a href="#installation-6">Installation</a></li> <li><a href="#troubleshooting-6">Troubleshooting</a></li> </ul></li> <li><a href="#reporting-issues">Reporting Issues</a></li> </ul> <h2><a id="icecat" class="anchor" href="#icecat">¶</a>Icecat</h2><p>Icecat is the fully free (as in freedom) fork of Firefox. It usually lags behind by quite a few versions (using the Icecat ESR version,) but otherwise is a safer Firefox. It comes by default with quite a few add-ons, including LibreJS (explicitly asking you if you want to run nonfree JS), Searxes' (which allows you to choose which CDNs to enable), and a few others. Icecat has settings to disable unsafe options, such as WebGL and WebRTC. LibreJS can also be a major issue for websites, but can be disabled. If you do not need Chromium or Firefox, consider using Icecat. </p> <p><strong>Homepage:</strong> <a href="https://www.gnu.org/software/gnuzilla/">www.gnu.org/software/gnuzilla</a> </p> <h3><a id="installation" class="anchor" href="#installation">¶</a>Installation</h3><p>Icecat is in the official repository, and has substitutes (no need to compile.) </p> <h3><a id="troubleshooting" class="anchor" href="#troubleshooting">¶</a>Troubleshooting</h3><p>No issues currently reported! </p> <h2><a id="firefox---strongnfstrong" class="anchor" href="#firefox---strongnfstrong">¶</a>Firefox - <strong>NF</strong></h2><p>Firefox is an open source browser from Mozilla. One of the most popular browsers for GNU/Linux, many forks of Firefox exist to improve it and make it more private (such as Icecat or Librewolf.) Permission is needed to use the Firefox name from Mozilla, as well as other restrictions, which make it a restrictive package, even if it is open source software. This bars it from being in the official repositories. </p> <p><strong>Homepage:</strong> <a href="https://www.mozilla.org/firefox">www.mozilla.org/firefox</a> </p> <h3><a id="installation" class="anchor" href="#installation">¶</a>Installation</h3><p>You can install Firefox through Nonguix, Nix, Flatpak, and even Docker. To install through Nix, please view the "Nix" page &lt;placeholder&gt; and install as you would any other program (either declarative or imperative.) </p> <p>The preferred way to install Firefox is through nonguix if you would like to keep your configurations in Scheme/Guix and are alright with compiling it, or Nix if you want a pre-compiled binary. </p> <h3><a id="troubleshooting" class="anchor" href="#troubleshooting">¶</a>Troubleshooting</h3><h4><a id="firefox-crashes-when-saving-a-file-or-uploading-a-file" class="anchor" href="#firefox-crashes-when-saving-a-file-or-uploading-a-file">¶</a>Firefox crashes when saving a file or uploading a file</h4><p>If you do not have a GTK theme installed, Firefox will crash when performing actions with a file explorer (and potentially more processes.) Install a GTK theme in the same profile as your Firefox install to prevent this issue. </p> <h4><a id="no-audio-in-browser" class="anchor" href="#no-audio-in-browser">¶</a>No audio in browser</h4><p>Sometimes Firefox does not connect to Pulseaudio; solution currently unknown. Restarting the browser has fixed this every time for me. This issue does not occur commonly at all. </p> <p>-Daniel </p> <h4><a id="no-webgl" class="anchor" href="#no-webgl">¶</a>No WebGL</h4><p>Firefox may not have any WebGL capabilities under certain combinations of installed packages or settings (such as WebRender.) No exact combination is documented that can replicate this, and I have yet to experience it again. </p> <p>-Daniel </p> <h2><a id="ungoogled-chromium" class="anchor" href="#ungoogled-chromium">¶</a>Ungoogled Chromium</h2><p>If you prefer Chromium to the Firefox family of browsers but want to maintain your safety and privacy, look no further. Ungoogled Chromium is "Google Chromium, sans integration with Google." Ungoogled Chromium has numerous patches to Chromium and enhancing features. </p> <p><strong>Homepage:</strong> <a href="https://github.com/Eloston/ungoogled-chromium">github.com/Eloston/ungoogled-chromium</a> </p> <h3><a id="installation" class="anchor" href="#installation">¶</a>Installation</h3><p>Ungoogled Chromium is in the official repository, and has substitutes (no need to compile.) </p> <h3><a id="troubleshooting" class="anchor" href="#troubleshooting">¶</a>Troubleshooting</h3><p>No issues currently reported! </p> <h2><a id="nyxt" class="anchor" href="#nyxt">¶</a>Nyxt</h2><p>Nyxt is a "keyboard-oriented, infinitely extensible web browser designed for power users." It is written in Common Lisp, and has keybindings for Emacs, vi, and CUA. Put simply, it is a "Common Lisp based Qutebrowser." Everything is configurable in Lisp, and the browser itself is based on buffers instead of tabs, and uses WebKit and WebEngine. It does not currently have support for WebExtensions, but is underway. </p> <p><strong>Homepage:</strong> <a href="https://nyxt.atlas.engineer/">nyxt.atlas.engineer</a> </p> <h3><a id="installation" class="anchor" href="#installation">¶</a>Installation</h3><p>Nyxt is in the official repository, and has substitutes (no need to compile.) </p> <h3><a id="troubleshooting" class="anchor" href="#troubleshooting">¶</a>Troubleshooting</h3><h4><a id="no-html-video" class="anchor" href="#no-html-video">¶</a>No HTML video</h4><p>Ensure that you have the following packages installed: </p> <pre><code class="scheme"><span class="org-string">"gst-libav"</span> <span class="org-string">"gst-plugins-bad"</span> <span class="org-string">"gst-plugins-base"</span> <span class="org-string">"gst-plugins-good"</span> <span class="org-string">"gst-plugins-ugly"</span> </code></pre> <h2><a id="qutebrowser" class="anchor" href="#qutebrowser">¶</a>Qutebrowser</h2><p>Qutebrowser is a keyboard and vi focused browser. It is written and configured in Python. It is similar to dwb and Nyxt. Tab and adblocking support is built in. </p> <p><strong>Homepage:</strong> <a href="https://qutebrowser.org/">qutebrowser.org</a> </p> <h3><a id="installation" class="anchor" href="#installation">¶</a>Installation</h3><p>Qutebrowser is in the official repository, and has substitutes (no need to compile.) </p> <h3><a id="troubleshooting" class="anchor" href="#troubleshooting">¶</a>Troubleshooting</h3><p>No issues currently reported! </p> <h2><a id="chromium---strongnfstrong" class="anchor" href="#chromium---strongnfstrong">¶</a>Chromium - <strong>NF</strong></h2><p>Chromium is an open-source browser from Google. Unless you need it, Ungoogled Chromium is recommended over regular Chromium to better protect your privacy and add enhancements. </p> <p><strong>Homepage:</strong> <a href="https://www.chromium.org/">www.chromium.org</a> </p> <h3><a id="installation" class="anchor" href="#installation">¶</a>Installation</h3><p>Chromium is not in the Guix repositories, but can be installed from Nix or Flatpak (might be possible through other methods.) </p> <h3><a id="troubleshooting" class="anchor" href="#troubleshooting">¶</a>Troubleshooting</h3><p>No issues currently reported! </p> <h2><a id="brave---strongnfstrong" class="anchor" href="#brave---strongnfstrong">¶</a>Brave - <strong>NF</strong></h2><p>Brave is a privacy-oriented Chromium-based browser. It features its own cryptocurrency, BAT, and has features such as IPFS built in. </p> <p><strong>Homepage:</strong> <a href="https://brave.com/">brave.com</a> </p> <h3><a id="installation" class="anchor" href="#installation">¶</a>Installation</h3><p>Brave is not in the Guix repositories, but can be installed through Nix or Snap. </p> <h3><a id="troubleshooting" class="anchor" href="#troubleshooting">¶</a>Troubleshooting</h3><p>No issues currently reported! </p> <h2><a id="reporting-issues" class="anchor" href="#reporting-issues">¶</a>Reporting Issues</h2><p>If you have any issues with these browsers, please open an issues at the wiki repository at GitHub: <a href="https://github.com/systemcrafters/wiki-site">github.com/systemcrafters/wiki-site</a>. Your issues help us solve and document them for everyone! </p> </div>

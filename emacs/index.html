GNU Emacs

<div><p>Refer to the video series and show notes on Emacs for more thorough "tutorial" style content that David has put together. Here are some quick jumping off points: </p> <h2><a id="emacs-essentials" class="anchor" href="#emacs-essentials">¶</a>Emacs Essentials</h2><p>This is a <a href="https://systemcrafters.net/emacs-essentials/">series</a> for true beginners and explains the fundamentals of Emacs. </p> <h2><a id="emacs-from-scratch" class="anchor" href="#emacs-from-scratch">¶</a>Emacs from scratch</h2><p>This is a <a href="https://systemcrafters.net/emacs-from-scratch/">series</a> geared towards crafting your custom Emacs configuration. Out of the box configurations like <a href="https://github.com/hlissner/doom-emacs">doom</a>, <a href="https://www.spacemacs.org/">spacemacs</a>, and <a href="https://github.com/bbatsov/prelude">prelude</a> are also good options to see how these "opinionated" setup the emacs environment so you can hack on your own setup more. </p> <h2><a id="emacss-help-system" class="anchor" href="#emacss-help-system">¶</a>Emacs's help system</h2><p>Emacs is a self documented editor, meaning with couple of key presses, you can find user guides, description of commands, functions, variables and more. Head over the <a href="/emacs/help-cheatsheet/">help cheatsheet</a> to learn more about this awesome documentation system. </p> <h2><a id="more-emacs-content" class="anchor" href="#more-emacs-content">¶</a>More Emacs Content</h2><p>There's a lot more content on the <a href="https://systemcrafters.net/videos">Videos</a> page. </p> <h2><a id="emacs-and-org-mode" class="anchor" href="#emacs-and-org-mode">¶</a>Emacs and org-mode</h2><p>org-mode is an outline markup format that helps you take notes, manage and schedule your TODOs, and a lot more. </p> <p>Here are some useful related packages: </p> <ul><li><a href="/emacs/org-roam/">org-roam</a>: A plain-text personal knowledge management system</li> <li>deft: Deft is an Emacs mode for quickly browsing, filtering, and editing directories of plain text notes, inspired by Notational Velocity</li> <li>org-super-agenda: This package lets you “supercharge” your Org daily/weekly agenda. The idea is to group items into sections, rather than having them all in one big list</li> </ul> <h2><a id="transitioning-to-emacs" class="anchor" href="#transitioning-to-emacs">¶</a>Transitioning to Emacs</h2><p>There is a good possibility that there is an overlap between the Emacs-curious and keyboard shortcut users. However, the Emacs keybidings are quite different than the Common User Access (CUA) that a lot of users are familiar with. <span class="org-builtin">:gin</span> <span class="org-string">"C-S-x"</span> #'simpleclip-cut <span class="org-comment-delimiter">;</span><span class="org-comment">Was: C-x chord</span> <span class="org-builtin">:gin</span> <span class="org-string">"C-S-c"</span> #'simpleclip-copy <span class="org-comment-delimiter">;</span><span class="org-comment">Was: C-x chord</span> <span class="org-builtin">:gin</span> <span class="org-string">"C-S-v"</span> #'clipboard-yank <span class="org-comment-delimiter">;</span><span class="org-comment">freezing on Ubuntu: 'simpleclip-paste ;Was: C-x chord</span> <span class="org-builtin">:gin</span> <span class="org-string">"C-z"</span> #'undo <span class="org-comment-delimiter">; </span><span class="org-comment">Was: enable Emacs state</span> <span class="org-builtin">:gin</span> <span class="org-string">"C-S-z"</span> #'redo <span class="org-comment-delimiter">;</span><span class="org-comment">Was: C-x chor</span> <span class="org-builtin">:gin</span> <span class="org-string">"C-&lt;tab&gt;"</span> #'switch-to-next-buffer <span class="org-comment-delimiter">;</span><span class="org-comment">Was: aya-create snippet</span> <span class="org-builtin">:gin</span> <span class="org-string">"C-S-&lt;tab&gt;"</span> #'switch-to-prev-buffer <span class="org-comment-delimiter">;</span><span class="org-comment">Was: C-x chord</span> <span class="org-builtin">:gin</span> <span class="org-string">"C-w"</span> #'kill-buffer <span class="org-comment-delimiter">;</span><span class="org-comment">Was: evil-window-map</span> <span class="org-builtin">:gin</span> <span class="org-string">"C-a"</span> #'mark-whole-buffer <span class="org-comment-delimiter">;</span><span class="org-comment">Was: doom/backward-to-bol-or-indent</span> ) <span class="org-comment-delimiter">;; </span><span class="org-comment">Save.
